构造函数

<constructor> Mirai(String server, String authKey, String qqNumber)

创建一个新的Mirai对象并准备与http api服务器连接。
服务器地址，验证密钥和操作QQ号分别由server，authKey和qqNumber指定。

详细方法

● VOID setAuthKey(String authKey)

设置Mirai对象连接http api服务器时发送的密钥。

注：只有在connect()之前设定才有效，connect()之后设定并不会更改已建立连接的Session的属性。

● VOID setSeverhost(String server)

设置Mirai对象连接http api服务器地址。

注：只有在connect()之前设定才有效，connect()之后设定并不会更改已建立连接的Session的属性。

● VOID setQQNumber(String qqNumber)

设置Mirai对象连接http api服务器后操作的QQ号。

注：只有在connect()之前设定才有效，connect()之后设定并不会更改已建立连接的Session的属性。

● VOID auth(String server, String authKey)

向http api发送认证请求。

Mirai.Session connect()

连接服务器并建立一个会话窗口。
若未指定服务器地址和验证密钥，将抛出Server host or authenticate key isnt set.异常。
若验证密钥错误，将抛出Authenticate key is invaild.异常。

构造函数

<constructor> Mirai.Session(String sessionId)

创建一个Session

详细方法

● String getSessionId()

返回当前Session的id(由http api分配)。

● VOID setSessionId(String sessionId)

指定当前Session的ID。

● VOID setAttributes(JSON attr)

设置当前Session属性。

● String getQQNumber()

返回当前Session操作的QQ号。

若已开始执行session循环验证线程或消息循环监听线程，则会自动更新配置并重启线程。

● VOID setMessageListener(Mirai.MessageListener listener)

设置消息监听器。

若已开始执行session循环验证线程或消息循环监听线程，则会自动更新配置并重启线程。

● VOID setVerifyThreadLoopInterval(Number interval)

设置session验证循环线程执行间隔(毫秒)。
默认是1500000毫秒(25分钟)。
一个Session在30分钟后未验证则http服务端自动销毁，所以请不要把此间隔设定在30分钟以上。

若已开始执行session循环验证线程或消息循环监听线程，则会自动更新配置并重启线程。

● VOID setListenThreadLoopInterval(Nubmer interval)

设置消息监听循环监听线程执行间隔(毫秒)。
默认是200毫秒(0.2秒)。
间隔时间越短，获取消息越及时，但是会占用更多内存。
间隔时间过长可能会漏接消息。

若已开始执行session循环验证线程或消息循环监听线程，则会自动更新配置并重启线程。

● VOID startVerifyThread()

开始执行session循环验证线程。

● VOID stopVerifyThread()

停止session循环验证线程。

● VOID startListen()

开始执行消息循环监听线程。

若未为该session设置消息监听器，则抛出Cannot start to listen: No listener set.异常。

● VOID stopListen()

停止消息循环监听线程。

● VOID release()

向服务端发送销毁session信号并立即释放当前session。

● Number sendFriendMessage(Number target, Mirai.MessageChain messageChain[, Number quoteId])

向QQ号为target的好友发送消息，参数quoteId是可选项，表示引用回复的消息ID。
若发送成功则返回消息ID，否则返回0。

● Number sendGroupMessage(Number target, Mirai.MessageChain messageChain[, Number quoteId])

向QQ群号为target的群发送消息，参数quoteId是可选项，表示引用回复的消息ID。
若发送成功则返回消息ID，否则返回0。

● Number sendTempMessage(Number target, Number from, Mirai.MessageChain messageChain[, Number quoteId])

向QQ号为target的群好友发送临时消息，from表示该用户来自的群的群号，参数quoteId是可选项，表示引用回复的消息ID。
若发送成功则返回消息ID，否则返回0。

● VOID recall(Number target)

撤回一条消息，target指定了要撤回的消息的ID。

● Mirai.MessageChain getCachedMessage(Number messageId)

通过消息id从消息缓存中返回一条消息的消息链，可以是群组消息或好友消息，但不能是事件。
若缓存中无此消息id对应的消息链，则返回一个新的无任何消息的消息链。

● VOID mute(Number group, Number target, Number time)

禁言一个群员，group指定要操作的群，target指定要禁言的成员，time指定禁言的时间(秒)。

消息类型构造方法

<constructor> Mirai.MessageType.Source(Number id, Number time)

源消息链首，id为消息ID，time为UNIX时间。

构建Source类型消息毫无意义，因为Source只作为群组消息ID的标记，而发送消息不需要消息ID(除了引用)。

<constructor> Mirai.MessageType.Quote(Number id, Number senderId, Number groupId, Mirai.MessageChain origin)

引用消息，id为消息ID，senderId为引用原文发送者QQ号，groupId为该消息QQ群的群号，origin为引用原文消息链。

构建Quote类型消息毫无意义，因为发送引用消息的方法是在Mirai.Session.sendGroupMessage的第三个参数传入要引用的消息ID。

<constructor> Mirai.MessageType.Plain(String text)

文字消息，text为文本内容。

<constructor> Mirai.MessageType.Image(String imageId, String url, String path)

图片消息，imageId为图片的UUID，url为图片的链接，path为图片本地链接。
三者可以任选一个参数，当同时有多个参数时，http api会按照imageId>url>path的优先级发送图片。
接收到的图片永远不可能会有path。

<constructor> Mirai.MessageType.FlashImage(String imageId, String url, String path)

闪照消息，用法和Mirai.MessageType.Image完全相同。

<constructor> Mirai.MessageType.Face(Number faceId, String name)

QQ表情，faceId为表情编号(优先级高于后面的name)，name为表情名称。

<constructor> Mirai.MessageType.At(Number target, String displayText)

@某人，taeget为被@的人的QQ号，displayText为@发送时显示的@文字。

<constructor> Mirai.MessageType.AtAll()

@全体成员。

<constructor> Mirai.MessageType.Xml(String xml)

XML消息，xml为XML消息内容。

<constructor> Mirai.MessageType.Json(String json)

JSON消息，json为JSON消息内容。

<constructor> Mirai.MessageType.App(String app)

APP消息，app为APP消息内容。

<constructor> Mirai.MessageType.Poke(Mirai.MessageTypeConst.PokeType.? name)

Poke(戳一戳)消息，name为Poke消息内容。

详细方法

● VOID static create(String path)

在path创建一个文件或目录。

● VOID static remove(String path)

删除在path的一个文件或目录。

● ? static read(String path, ? mode)

读取在path的文件。
如果mode为FileIO.STRING，则返回文件内容。
如果mode为FileIO.STREAM，则返回文件输入流。

● VOID static writeString(String path, String string, Boolean isCover)

写入字符串到path的文件。
isCover设定是否在文件已存在的情况下覆盖文件内容。

● VOID static writeStream(String path, java.io.InputStream inputStream, Boolean isCover, Boolean isInputStreamClose)

写入字符串到path的文件。
isCover设定是否在文件已存在的情况下覆盖文件内容。

● VOID static appendString(String path, String appendString)

追加字符串到path的文件。
